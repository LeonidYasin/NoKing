# Конституция DAO NoKing

**Версия:** 1.0
**Принята:** первыми участниками
**Легитимность:** подтверждается смарт-контрактом

---

## 1. Кто управляет NoKing

**DAO (Децентрализованная Автономная Организация)** — коллективный орган управления, состоящий из активных участников системы:

- Любой участник, совершивший не менее 3 успешных обменов.
- Разработчик, сделавший не менее 2 принятых Pull Request в репозиторий.
- Арбитр, рассмотревший не менее 5 споров.

Никаких «советов директоров», «главных архитекторов», «лидеров сообщества».  
Все решения принимаются коллективно и открыто.

---

## 2. Принципы голосования

1. **Один участник — один голос** (не по балансу, не по статусу).
2. **Защита от ботов** — право голоса получают только кошельки, совершившие реальные действия (обмен, PR, арбитраж).
3. **Кворум** — решение считается принятым, если в голосовании участвовало не менее 40% всех активных участников.
4. **Порог принятия:**
   - Обычные вопросы (комиссия, косметика) — 50% + 1
   - Принципиальные вопросы (изменение кодекса, архитектуры) — 75%
   - Критические вопросы (замена протокола, форк) — 90%

---

## 3. Категории голосований

| Тип | Что решаем | Порог |
|-----|------------|-------|
| Финансовый | Комиссия протокола, распределение фондов | 50% |
| Технический | Обновление контрактов, интеграции, стек | 50% |
| Кодекс | Изменение правил, принципов | 75% |
| Арбитражный | Отмена/подтверждение решения арбитра | 75% |
| Состав DAO | Приём/исключение участников DAO | 75% |
| Форк | Переход на другой блокчейн, смена архитектуры | 90% |

---

## 4. Инструменты DAO

- **Snapshot** — для голосований (офчейн, газ-фри)
- **Gnosis Safe** — для управления казной (мультиподпись)
- **Смарт-контракт** — для автоматического исполнения решений
- **Форум / Telegram** — для обсуждений перед голосованием

---

## 5. Казённый фонд DAO

- Комиссия протокола (0.5–1%) поступает в казну DAO.
- Казной управляет мультиподпись (5 из 9 ключей, избранных сообществом).
- Расходы:
  - Аудит смарт-контрактов
  - Развитие открытого интерфейса
  - Поощрение разработчиков (гранты)
  - Оплата серверов (если нужны централизованные элементы)
- Прозрачная отчётность публикуется ежеквартально.

---

## 6. Права и обязанности члена DAO

| Что даёт голос | Обязанности |
|----------------|-------------|
| Участие в голосованиях | Знать предложения перед голосованием |
| Предложение изменений | Быть активным (голосовать не реже 1 раза в месяц) |
| Инициация арбитража | Соблюдать кодекс чести |
| Влияние на развитие | Участвовать в обсуждениях (форум / чат) |

---

## 7. Как исключить участника из DAO

1. Предложение об исключении вносит любой активный участник.
2. Причина: нарушение кодекса или неучастие в работе DAO (пропуск 5 голосований подряд).
3. Решение принимается голосованием (75%).
4. Исключённый может подать апелляцию через 6 месяцев.
